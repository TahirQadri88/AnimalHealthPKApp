Build a full-featured, mobile-first Progressive Web App (PWA) for managing a

useLiveCollection HOOK (copy exactly):
  function useLiveCollection(collectionName) {
    const [data, setData] = React.useState([]);
    useEffect(() => {
      const unsubscribe = onSnapshot(collection(db, collectionName), (snapshot) => {
        const items = [];
        snapshot.forEach((d) => items.push(d.data()));
        setData(items.sort((a, b) => (a.id > b.id ? 1 : -1)));
      }, (error) => { console.error('Error fetching ' + collectionName + ':', error); });
      return () => unsubscribe();
    }, [collectionName]);
    return data;
  }

    try {
      if (!e || !e.postData) throw new Error('No POST data received');
      var payload = JSON.parse(e.postData.contents);
      var folder = payload.folderId
        ? DriveApp.getFolderById(payload.folderId)
        : DriveApp.getRootFolder();
      var existing = folder.getFilesByName(payload.filename);
      while (existing.hasNext()) existing.next().setTrashed(true);
      folder.createFile(payload.filename, payload.content, MimeType.PLAIN_TEXT);
      var keep = payload.keepCount || 4;
      var files = [];
      var iter = folder.searchFiles('title contains "Backup_"');
      while (iter.hasNext()) files.push(iter.next());
      files.sort(function(a, b) { return a.getDateCreated() - b.getDateCreated(); });
      while (files.length > keep) files.shift().setTrashed(true);
      return ContentService
        .createTextOutput(JSON.stringify({ ok: true }))
        .setMimeType(ContentService.MimeType.JSON);
    } catch (err) {
      return ContentService
        .createTextOutput(JSON.stringify({ error: err.message }))
        .setMimeType(ContentService.MimeType.JSON);
    }
  }
  function doGet(e) {

================================================================================
  FEATURE 10 — CUSTOMER LEDGER CALCULATIONS
================================================================================

getCustomerLedger(customerId) function must:
1. Start with openingBalance from customer record (positive = they owe money).
2. Sort all transactions (invoices, creditNotes, payments) by date ascending.
3. For each invoice (status Billed or Dispatched): add total to running balance (debit).
4. For each credit note (status CreditNote): subtract total from running balance (credit).
5. For each payment: subtract amount from running balance (credit).
6. Return array of ledger rows with: { type, id, date, description, debit, credit, balance }.

getCustomerBalance(customerId):
  Returns final balance (positive = customer owes, negative = credit).

================================================================================

- Invoices get a seqNum: find max seqNum across all invoices (excluding CN- and DN- ids) + 1.
- Dispatch notes use same seqNum sequence.
- Credit notes get their own sequence: find max seqNum across CN- invoices + 1.

================================================================================
  CRITICAL BUSINESS LOGIC RULES
================================================================================

1. PROFIT CALCULATION:
   Gross Profit = (salePrice - costPrice) × qty, summed per item.
   Delivery charge (deliveryBilled) is revenue but NOT included in profit.
   Transport expense is deducted from profit but NOT shown to customer on invoice.
   Formula: grossProfit = items.reduce((sum, item) => sum + (item.rate - item.costPrice) * item.qty, 0)

2. CREDIT NOTES IN ANALYTICS:
   When a credit note exists, subtract from BOTH revenue AND cost AND profit.
   Use actual item costPrice from credit note — not a revenue-based estimate.
   Adjust: byCompany, byCustomer, bySalesperson, byCity, byArea, byType,
           monthlyData, dailyBreakdown, and top-line KPIs.

3. RECEIVABLES:
   Outstanding = sum of all customer balances where balance > 0.
   Do NOT include customers with credit (negative) balance in receivables total.

4. VEHICLE TYPES AND RIDERS:
   When a vehicleType with requiresRider=true is selected on an invoice:
     - Show rider search/selection filtered by that vehicleType.
   When a vehicleType with requiresRider=false is selected (and name != 'Self-Pickup'):
     - Show transport company + bilty number fields.
   Rider forms only show vehicleTypes where requiresRider=true.

5. INVOICE TOTAL:
   total = sum(item.qty * item.rate) - discount + deliveryBilled
   transportExpense is stored but NOT added to total (internal cost only).

  CONTEXT PATTERN — CRITICAL
================================================================================

All state and functions are defined in the main App() function and passed via
a single AppContext. Components access what they need via useContext(AppContext).

Every component that uses a collection from Firestore must destructure it from
useContext(AppContext). If you add a new collection, you MUST:
  1. Add useLiveCollection('newCollection') in App()
  2. Add it to the ctx object passed to AppContext.Provider
  3. Add it to every component's useContext destructure that needs it
  4. Add it to buildBackupObj()
  5. Add it to the auto-backup useEffect cols object
  6. Add it to the downloadBackup function

Failing any of these steps causes "X is not defined" runtime errors.

  BACKUP SYSTEM — COMPLETE IMPLEMENTATION

buildBackupObj() returns:
{
  exportedAt: new Date().toISOString(),
  collections: {
    app_users, appSettings: [appSettings], companies, products,
    customers, invoices, expenses, expenseCategories, payments,
    riders, cities, areas, customerTypes, vehicleTypes
  }
}

Firebase backup: splits into documents '{date}_{col}' in 'backups' collection.
  (Avoids Firestore 1MB per-document limit by keeping collections separate.)

Google Drive backup: sends full JSON in ONE file (no size limit on Drive).
  Filename: '{AppName}_Backup_{YYYY}-W{NN}.json' (ISO week number).
  Keeps last 4 weekly files (Apps Script handles rotation).

Auto-backup: runs once per session using a React.useRef flag.
  Checks lastBackupAt / lastDriveBackupAt against configured frequency.
  Runs both Firebase and Drive backups independently if each is due.

Restore: reads backup JSON, loops collections, writes each doc via saveToFirebase.
  Shows confirmation dialog before overwriting. Shows progress count on completion.

================================================================================
  KNOWN PITFALLS — READ BEFORE BUILDING
================================================================================

1. TAILWIND IN PRINT WINDOWS:
   Never use buildHtmlDoc() without injecting live CSS. The print window has no
   access to Tailwind's compiled CSS unless you inject it from document.styleSheets.
   ALL formats (not just thermal) need CSS injection.

2. HTML-TO-IMAGE VIEWPORT REQUIREMENT:
   html-to-image uses SVG foreignObject and CANNOT render elements at left:-9999px.
   Always clone the element and position it at top:0 left:0 with z-index:-1 (behind
   the app UI) before capturing. Remove after capture.

3. PDF SCALE:
   Always use scale:3 for html2canvas in html2pdf. scale:2 produces noticeably
   blurry output on high-DPI screens. scale:3 gives crisp text.

4. APPS SCRIPT CORS:
   Do NOT use fetch with mode:'no-cors' for the Google Drive relay.
   Apps Script public web apps return Access-Control-Allow-Origin:*.
   Using no-cors makes it impossible to read errors from the response.
   Use Content-Type:'text/plain' (a "simple" header) to avoid preflight,
   then read the JSON response normally.

5. VEHICLETYPE SEEDING RACE CONDITION:
   vehicleTypes starts as [] before Firestore responds (same as genuinely empty).
   Only seed when BOTH: appSettings.id exists (proves Firestore responded) AND
   vehicleTypes.length === 0. Use a React.useRef flag to prevent re-seeding.

6. CREDIT NOTE ANALYTICS:
   A common bug is subtracting only revenue from credit notes but not adjusting
   cost and profit independently. You must subtract the actual costPrice * qty
   for each returned item, separately from the revenue subtraction.

7. CONTEXT DESTRUCTURING:
   Every time you add a new collection, audit ALL useContext(AppContext) calls in
   ALL components and add the new collection to every component that needs it.
   Missing even one causes a "X is not defined" crash in that component.

8. FIRESTORE DOCUMENT SIZE:
   Firestore has a 1MB per-document limit. The backup system avoids this by
   storing each collection as a separate document. Never try to store all
   collections in one Firestore document.

9. SEQUENTIAL NUMBERS:
   Calculate seqNum at save time from max existing — never store a counter.
   Separate sequences for invoices and credit notes (CN-).

10. MOBILE SCROLL:
    Admin Settings and any long scrollable views must have pb-28 (padding-bottom)
    so content is reachable above the fixed bottom navigation bar.
